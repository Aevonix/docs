ClawColony provides an encrypted credential vault for storing secrets, API keys, and sensitive configuration. All credentials are encrypted at rest with AES-256-GCM.

## Encryption

| Property | Value |
| -------- | ----- |
| Algorithm | AES-256-GCM |
| Key derivation | Argon2id |
| Key storage | OS keychain or encrypted file |
| Nonce | Unique per encryption operation |

## Storing Credentials

```bash
# Store a credential
clawcolony secrets set OPENAI_API_KEY

# You'll be prompted for the value (not echoed)
Enter value for OPENAI_API_KEY: ****
Credential stored successfully.
```

Or provide inline (less secure, visible in shell history):

```bash
clawcolony secrets set OPENAI_API_KEY --value "sk-..."
```

## Retrieving Credentials

```bash
# Get a credential
clawcolony secrets get OPENAI_API_KEY

# Output is masked by default
OPENAI_API_KEY: sk-...****
```

For programmatic access:

```bash
# Get raw value (careful with shell history)
clawcolony secrets get OPENAI_API_KEY --raw
```

## Deleting Credentials

```bash
clawcolony secrets delete SLACK_BOT_TOKEN
```

## Credential Types

The vault recognizes credential types for enhanced security:

| Type | Validation | Notes |
| ---- | ---------- | ----- |
| `api_key` | Format check | Provider-specific patterns |
| `token` | None | Generic tokens |
| `password` | Strength check | Warns on weak passwords |
| `certificate` | PEM validation | Checks validity |
| `private_key` | Key validation | Ensures proper format |

Specify type explicitly:

```bash
clawcolony secrets set DB_PASS --type password
```

## Storage Locations

| Platform | Primary Storage | Fallback |
| -------- | --------------- | -------- |
| macOS | Keychain | Encrypted file |
| Linux | Secret Service / libsecret | Encrypted file |
| Windows | Credential Manager | Encrypted file |

Encrypted file location: `~/.clawcolony/credentials/vault.enc`

## Master Password

If using file-based storage:

```bash
# Set master password
clawcolony secrets init

Enter master password: ****
Confirm master password: ****
Vault initialized.
```

The master password is required to:
- Add new credentials
- Retrieve credential values
- Change vault settings

## Programmatic Access

In TypeScript code:

```typescript
import { SecretStore } from "@clawcolony/colony";

const secrets = new SecretStore(config);

// Get a credential
const apiKey = await secrets.get("OPENAI_API_KEY");

// Set a credential
await secrets.set("NEW_KEY", "value", { type: "api_key" });

// Check if exists
if (await secrets.has("OPTIONAL_KEY")) {
  // Use it
}
```

## Environment Variable Integration

Credentials can be loaded from environment variables:

```json
{
  "colony": {
    "secrets": {
      "envPrefix": "CLAWCOLONY_",
      "loadFromEnv": true
    }
  }
}
```

With this config, `CLAWCOLONY_OPENAI_API_KEY` env var becomes `OPENAI_API_KEY` in the vault.

## Rotation

Rotate credentials safely:

```bash
# Set new value, keeps old as backup
clawcolony secrets rotate OPENAI_API_KEY

Enter new value: ****
Old value backed up. New value active.
```

Rollback if needed:

```bash
clawcolony secrets rollback OPENAI_API_KEY
```

## Security Best Practices

1. **Use OS keychain**: More secure than file-based storage
2. **Strong master password**: 20+ characters, mixed case, symbols
3. **Rotate regularly**: Set calendar reminders
4. **Audit access**: Check `clawcolony security audit` for credential access
5. **Backup securely**: Export vault to encrypted offline storage

## Export/Import

```bash
# Export (encrypted)
clawcolony secrets export --output vault-backup.enc

# Import
clawcolony secrets import vault-backup.enc
```

Export is encrypted with your master password.

## Configuration

```json
{
  "colony": {
    "secrets": {
      "storage": "keychain",
      "fallbackToFile": true,
      "encryptionIterations": 100000,
      "rotationReminderDays": 90
    }
  }
}
```
