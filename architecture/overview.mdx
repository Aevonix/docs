ClawColony is built as a stack of independent modules that communicate through a typed event bus. Each module implements the `CognitionPlugin` interface, providing a consistent pattern for initialization, migration, and shutdown.

## Module Categories

| ------ | ------- | --------- |

| Gossip | SWIM membership protocol, peer discovery | `swim-protocol.ts` |
| Mesh | Node coordination, task routing | `mesh-manager.ts` |
| Chain | Block production, consensus, state machine | `chain-manager.ts` |

## Event Bus

Modules communicate through a typed event bus. Events are strongly typed with TypeScript interfaces.

```typescript
interface ColonyEvent {
  type: string;
  payload: unknown;
  timestamp: string;
  source: string;
}

// Example: Memory extracted event
interface MemoryExtractedEvent extends ColonyEvent {
  type: "memory:extracted";
  payload: {
    memoryId: string;
    content: string;
    importance: number;
    entities: string[];
  };
}
```

Modules subscribe to events they care about:

```typescript
eventBus.subscribe("memory:extracted", async (event) => {
  // Update knowledge graph
  await this.graphManager.addMemory(event.payload);
});
```

## Database Schema

All data lives in SQLite. Each module defines its own tables via `getMigrationSQL()`:

```typescript
export class MemoryPlugin implements CognitionPlugin {
  readonly name = "memory";

  getMigrationSQL(): string {
    return `
      CREATE TABLE IF NOT EXISTS memories (
        id TEXT PRIMARY KEY,
        content TEXT NOT NULL,
        importance REAL NOT NULL DEFAULT 0.5,
        created_at TEXT NOT NULL,
        recall_count INTEGER NOT NULL DEFAULT 0
      );

      CREATE INDEX IF NOT EXISTS idx_memories_importance
        ON memories(importance DESC);
    `;
  }
}
```

Key SQLite features used:
- **WAL mode**: Concurrent reads during writes
- **FTS5**: Full-text search for memory content
- **Recursive CTEs**: Knowledge graph traversal
- **JSON functions**: Flexible metadata storage

## Configuration

All modules read from a unified config at `~/.clawcolony/clawcolony.json`:

```json
{
  "gateway": {
    "port": 18791,
    "bind": "127.0.0.1"
  },
  "colony": {
    "memory": { /* memory config */ },
    "router": { /* model router config */ },
    "network": { /* gossip/mesh config */ },
    "chain": { /* blockchain config */ }
  }
}
```

See [Configuration Reference](/reference/configuration) for all options.
