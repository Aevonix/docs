The ClawColony gateway exposes HTTP API endpoints for programmatic access.

## Base URL

```
http://localhost:18791
```

## Authentication

Most endpoints require authentication via bearer token:

```bash
curl -H "Authorization: Bearer <token>" http://localhost:18791/colony/v1/status
```

## Health Endpoints

### GET /health

Basic liveness check.

```bash
curl http://localhost:18791/health
```

Response:
```json
{
  "status": "healthy",
  "version": "1.0.0"
}
```

## Memory Endpoints

### GET /colony/v1/memories

List memories.

Query parameters:
- `limit`: Max results (default: 20)
- `offset`: Pagination offset
- `type`: Filter by type

```bash
curl "http://localhost:18791/colony/v1/memories?limit=10"
```

### GET /colony/v1/memories/:id

Get specific memory.

```bash
curl http://localhost:18791/colony/v1/memories/mem-123
```

### POST /colony/v1/memories/search

Search memories.

```bash
curl -X POST http://localhost:18791/colony/v1/memories/search \
  -H "Content-Type: application/json" \
  -d '{"query": "project status", "limit": 10}'
```

## Node Endpoints

### GET /colony/v1/nodes

List nodes.

```bash
curl http://localhost:18791/colony/v1/nodes
```

### POST /colony/v1/nodes/invite

Generate invite token.

```bash
curl -X POST http://localhost:18791/colony/v1/nodes/invite \
  -H "Content-Type: application/json" \
  -d '{"role": "drone", "expiresIn": "7d"}'
```

### DELETE /colony/v1/nodes/:id

Revoke node.

```bash
curl -X DELETE http://localhost:18791/colony/v1/nodes/node-e5f6 \
  -H "Content-Type: application/json" \
  -d '{"reason": "Decommissioned"}'
```

## Message Endpoints

### POST /colony/v1/messages

Send message to colony.

```bash
curl -X POST http://localhost:18791/colony/v1/messages \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello, colony!", "channel": "api"}'
```

Response:
```json
{
  "messageId": "msg-123",
  "response": "Hello! How can I help you today?",
  "memories_used": 3
}
```

## WebSocket

### ws://localhost:18792

Real-time communication via WebSocket.

Connect:
```javascript
const ws = new WebSocket("ws://localhost:18792");

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log(message);
};

ws.send(JSON.stringify({
  type: "message",
  content: "Hello!"
}));
```

Message types:
- `message`: User message
- `status`: Status update
- `memory_update`: Memory changed
- `node_event`: Node joined/left

## Error Responses

Errors follow consistent format:

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Memory not found",
    "details": {}
  }
}
```

HTTP status codes:
- `400`: Bad request
- `401`: Unauthorized
- `403`: Forbidden
- `404`: Not found
- `429`: Rate limited
- `500`: Internal error

## Rate Limiting

Endpoints are rate-limited:
- Default: 100 requests/minute
- Search: 20 requests/minute
- Write: 50 requests/minute

Rate limit headers:
- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`
