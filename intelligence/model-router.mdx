The Model Router selects the appropriate model for each task based on complexity, cost, and capabilities. It uses a 4-layer routing system with 5 model tiers.

## Model Tiers

| Tier | Name | Example Models | Use Case |
| ---- | ---- | -------------- | -------- |
| 1 | None | Local/Ollama (phi3) | Simple lookups, formatting |
| 2 | Minimal | Claude Haiku | Quick responses, simple tasks |
| 3 | Standard | Claude Sonnet | Most tasks, good balance |
| 4 | Enhanced | Claude Opus | Complex reasoning, important tasks |
| 5 | Maximum | Claude Opus + tools | Critical tasks, full capability |

## 4-Layer Routing

```
┌─────────────────────────────────────────────────────────────┐
│                    Model Router                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Layer 1: Task-Type Routing                                 │
│  ─────────────────────────                                  │
│  Match task type to minimum required tier                   │
│                                                              │
│  Layer 2: Complexity Assessment                             │
│  ─────────────────────────────                              │
│  Analyze input to determine actual complexity               │
│                                                              │
│  Layer 3: Cascade Strategy                                  │
│  ────────────────────────                                   │
│  Try lower tier first, escalate if needed                   │
│                                                              │
│  Layer 4: Feedback Integration                              │
│  ──────────────────────────                                 │
│  Learn from past routing decisions                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Layer 1: Task-Type Routing

Default tier assignments by task type:

| Task Type | Default Tier |
| --------- | ------------ |
| Greeting/small talk | 1 (None) |
| Factual lookup | 2 (Minimal) |
| Summarization | 2 (Minimal) |
| Code generation | 3 (Standard) |
| Analysis | 3 (Standard) |
| Complex reasoning | 4 (Enhanced) |
| Critical decisions | 5 (Maximum) |

### Layer 2: Complexity Assessment

Factors that increase tier:

| Factor | Tier Increase |
| ------ | ------------- |
| Long input (>10k tokens) | +1 |
| Multiple entities | +1 |
| Technical domain | +1 |
| Ambiguous request | +1 |
| User flagged important | +1 |

### Layer 3: Cascade Strategy

Start low, escalate if needed:

1. Route to initial tier
2. Check response quality
3. If below threshold, retry with higher tier
4. Track escalation patterns

```typescript
interface CascadeConfig {
  enabled: boolean;
  qualityThreshold: number;    // 0.0 to 1.0
  maxEscalations: number;      // Max tier jumps
  budgetCapMonthly: number;    // Cost limit
}
```

### Layer 4: Feedback Integration

Learn from routing decisions:

- Track success rate per task-type/tier combo
- Adjust default tiers based on outcomes
- Identify under/over-provisioning patterns

## Budget Management

The router tracks costs and enforces limits:

```typescript
interface BudgetStatus {
  monthlyBudget: number;
  spent: number;
  remaining: number;
  daysRemaining: number;
  projectedOverage: number;
  recommendations: string[];
}
```

When approaching budget limits:
1. Cascade becomes more aggressive (start lower)
2. Tier 4/5 tasks queue for batch processing
3. User notified of budget status

## Router Interface

```typescript
interface ModelRouter {
  /** Route a task to appropriate model */
  route(task: Task): Promise<RoutingDecision>;

  /** Get current budget status */
  getBudgetStatus(): BudgetStatus;

  /** Override routing for specific task */
  override(taskId: string, tier: number): void;

  /** Get routing statistics */
  getStats(): RoutingStats;
}

interface RoutingDecision {
  tier: number;
  model: string;
  estimatedCost: number;
  reasoning: string;
  cascadeAllowed: boolean;
}
```
